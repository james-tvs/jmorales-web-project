# Final Project: GreenTech Solutions Web Portal

## Project Description
This web portal is designed for GreenTech Solutions, an environmental sustainability technology startup. The website educates small-to-medium businesses (SMBs) and eco-conscious individuals about green computing devices, smart recycling systems, and carbon tracking platforms. It balances intuitive aesthetics with rigorous programmatic structural principles.

## Technologies Used
- **Sematic HTML5** for structured accessibility layouts.
- **CSS3** via external style documentation containing responsive breakpoint rules.
- **JavaScript** handling hamburger layouts and form verification modules.

## Setup and Installation Instructions
1. Download or clone this project repository folder to your local computer.
2. Make sure all file paths follow the designated directory format (`/css`, `/js`, and `/images`).
3. Add custom placeholder target files (`hero.jpg`, `device.jpg`, `recycling.jpg`, `software.jpg`) inside your local `/images` path location.
4. Launch the `index.html` file using any modern browser engine (Chrome, Safari, Edge) to preview the complete system workspace locally.

## Fulfillment of Requirements Checklist
- **Theme:** Tailored directly to GreenTech Solutions client guidelines with a persistent global navigation system cross-linking internal structures seamlessly.
- **HTML:** Employs explicit structural structural markup parameters (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`) using 5+ tag categories (including data `<table>` rows and entry `<form>` layouts).
- **CSS:** Handled independently using an external sheet framework controlling deep visual themes and layout properties.
- **JavaScript:** Controls dynamic navigation drawer expansions along with comprehensive submission parameter checking loops on contact layouts.
- **Mobile Development:** Integrated responsive CSS grid layouts scaling through flex variations and strict media criteria below 768px bounds.
- **Web Accessibility:** Achieves deep contrast safety scores alongside comprehensive descriptive definitions across all targeted resource visual variables using explicitly labeled `alt` identifiers.
